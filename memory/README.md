# Memory 目录说明

这个目录用于存放 `OwnitAgent` 的记忆内存。当前设计参考
[`docs/context-management-design.md`](file:///d:/Code/Agents/demo_agents/docs/context-management-design.md)，
采用 `L0-L4` 分层方式组织长期记忆和历史归档。

## 分层概览

| 层级 | 位置 | 当前对应文件/目录 | 作用 |
|------|------|-------------------|------|
| `L0` | 记忆元规则层 | `memory_management_sop.md` | 规定“什么信息可以写入记忆、什么信息不能写入记忆” |
| `L1` | 极简索引层 | `global_mem_insight.txt` | 存放高频规则、关键词索引、记忆导航入口 |
| `L2` | 环境事实层 | `global_mem.txt` | 存放跨会话稳定的环境事实、固定配置、路径等 |
| `L3` | 任务级记忆层 | `task_sops/`、`tools/` | 存放专项 SOP、长期复用脚本、关键避坑记录 |
| `L4` | 历史归档层 | `L4_raw_sessions/` | 存放历史会话原始归档和后续压缩产物 |

## 当前文件与目录说明

### `memory_management_sop.md`

- 所属层级：`L0`
- 作用：记忆管理总规则
- 当前用途：
  - 约束只有“执行验证过”的信息才能写入记忆
  - 禁止写入推理猜测、计划草稿、易变状态
  - 规定 `L1/L2/L3` 各自应该存什么

这是写入任何记忆前应该先遵守的文件。

### `global_mem_insight.txt`

- 所属层级：`L1`
- 作用：极简索引
- 当前用途：
  - 放最短规则提示
  - 告诉模型去哪里找更详细的记忆

这个文件应该尽量短，不写长篇细节，主要负责“提醒模型某类记忆存在”。

### `global_mem.txt`

- 所属层级：`L2`
- 作用：环境事实库
- 当前用途：
  - 记录项目级稳定信息
  - 记录固定路径、常量、配置事实

这里适合放跨会话仍然成立的事实，不适合放会频繁变化的信息。

### `task_sops/`

- 所属层级：`L3`
- 作用：任务级 SOP 目录
- 当前状态：已创建，后续用于存放专项操作指南
- 适合内容：
  - 某类任务的固定流程
  - 某个场景的关键前置条件
  - 反复踩坑后沉淀下来的操作规则

### `tools/`

- 所属层级：`L3`
- 作用：长期复用脚本/工具目录
- 当前状态：已创建，后续用于存放高复用脚本
- 适合内容：
  - 不希望每次都重新推理的处理逻辑
  - 稳定复用的辅助脚本

### `sessions/`

- 作用：预留给当前项目的会话快照或摘要记录
- 当前状态：已创建，当前实现还没有完整归档策略
- 定位：
  - 这是“当前项目的运行期辅助目录”
  - 不直接归类为 `L1/L2/L3/L4` 的核心文件，但服务于记忆系统扩展

### `L4_raw_sessions/`

- 所属层级：`L4`
- 作用：历史会话归档目录
- 当前状态：已创建，预留给后续会话原始记录、压缩和归档
- 后续可能存放：
  - 原始会话文本
  - 压缩归档文件
  - 历史摘要整理结果

## 写入原则

当前 `memory/` 目录遵循以下规则：

1. 只有工具调用成功验证过的信息，才允许写入记忆。
2. 不写入易变状态，例如：
   - 时间戳
   - PID
   - 临时 Session ID
   - 当前瞬时状态
3. `L1` 只写极简索引，不写长篇细节。
4. `L2` 只写稳定事实。
5. `L3` 才写专项 SOP 和长期复用脚本。

## 当前实现状态

当前项目已经接入的记忆能力包括：

- 启动时自动创建 `memory/` 骨架
- 系统提示词会读取 `L1` 和 `L2`
- 提供了记忆相关工具：
  - `read_memory`
  - `write_memory`
  - `update_working_checkpoint`

尚未完整实现的部分：

- `L3` 目录内的真实 SOP/工具沉淀
- `L4` 的原始会话压缩与归档流程
- 更完整的历史摘要归档链路
